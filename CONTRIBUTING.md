# 🤝 Contributing to ZeroMark

Thank you for your interest in contributing to **ZeroMark**! Contributions, issues, and feature requests are welcome.

## 📋 How to Contribute

### 1. Fork the Repository

Click the **Fork** button at the top right of the [ZeroMark repository](https://github.com/tushar-infosec/ZeroMark).

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/ZeroMark.git
cd ZeroMark
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Your Changes

- Keep your changes focused and minimal
- Follow the existing code style
- Test your changes in multiple browsers

### 5. Commit Your Changes

```bash
git add .
git commit -m "Add: brief description of your change"
```

### 6. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 7. Open a Pull Request

Go to the original repository and click **New Pull Request**.

---

## 📝 Guidelines

- **Keep it simple** — ZeroMark is intentionally lightweight with zero dependencies
- **No external libraries** — This project uses vanilla HTML, CSS, and JavaScript
- **Test across browsers** — Ensure compatibility with Chrome, Firefox, Edge, and Safari
- **Write clear commit messages** — Use prefixes like `Add:`, `Fix:`, `Update:`, `Remove:`

## 🐛 Reporting Bugs

If you find a bug, please [open an issue](https://github.com/tushar-infosec/ZeroMark/issues) with:
- A clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Browser and OS information

---

Thank you for helping make ZeroMark better! 💚
